Table-Filling Algorithm (Myhill-Nerode)

DFA minimization finds the smallest possible DFA that accepts the same language. The key idea: two states are distinguishable if there exists some input string that leads to an accept state from one but not the other.

States that are indistinguishable (equivalent) can be safely merged — they behave identically for all possible future inputs.

Concept Definition
Distinguishable States p, q are distinguishable if ∃ string w such that exactly one of δ(p,w), δ(q,w) is an accept state
Indistinguishable States p, q are indistinguishable (equivalent) if for ALL strings w, δ(p,w) and δ(q,w) are both accepting or both rejecting
Minimization Merge all groups of mutually indistinguishable states into single states

Initial Partition — Accepting vs Non-Accepting

The first and most fundamental rule: an accepting state and a non-accepting state are always distinguishable — the empty string ε accepts from one but rejects from the other.

So we begin by marking all pairs (accept, non-accept) as distinguishable.

Table-Filling Iterations

We iteratively mark more pairs as distinguishable. For each unmarked pair (p, q) and each symbol a: if δ(p,a) and δ(q,a) lead to a distinguishable pair, then (p,q) is also distinguishable. Repeat until no new pairs can be marked.

Final Distinguishability Matrix

The completed table. ✗ Distinguished means the pair is distinguishable (cannot be merged). ≡ Equivalent means the pair can be merged.

Equivalence Classes — Groups to Merge

Each group of mutually indistinguishable states forms one equivalence class. Each class becomes a single state in the minimized DFA.
