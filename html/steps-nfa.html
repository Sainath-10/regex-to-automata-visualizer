Stage 1: Recursive Construction Principles
From Grammar to Machine

A Regular Expression can be converted into an equivalent NFA by recursively building basic building blocks (basis) and then combining them using inductive rules (union, concatenation, and closure).

This approach ensures that for any valid expression, there is a corresponding machine that accepts exactly the same language.

The resulting NFA has exactly one start state and one accept state, with ε (epsilon) transitions used to connect fragments together.

Key properties of the generated NFA:

PropertyDetail
Single start stateExactly one state with no incoming transitions
Single accept stateExactly one final/accepting state
ε-transitionsUsed freely to connect sub-NFAs without consuming input
No accept-state outgoingThe accept state has no outgoing transitions
Stage 2: Syntactic Preparation

Before construction begins, we must parse the expression to handle operator precedence (like parentheses and Kleene Star). The first step is classifying each character as either a literal symbol or a structural operator.

ℹ

Concatenation is implicit — there is no explicit concatenation character in the regex. The parser inserts a hidden · operator where two tokens are juxtaposed and concatenation is meant.

Stage 3: Resolving Operator Precedence

Languages usually treat concatenation as an implicit operation. To make the conversion logical, we insert explicit concatenation operators · between tokens wherever they are juxtaposed.

Stage 4: Establishing Order of Operations

We convert the expression to postfix notation. This removes the need for parentheses and clearly defines which sub-expressions are combined first. This reflects the order in which we build our NFA fragments.

Stage 5: Inductive Fragment Rules

For each operation in the postfix stream, we apply a specific conversion rule. Symbols create a basis (two states and a transition), while operators combine existing segments into larger structures.

Symbol / Literal
Single character a
→ q₀ ─a→ q₁
