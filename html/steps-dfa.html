The Principle of DFA Construction
State-Set Equivalence

An NFA can be in multiple states at once. To build a deterministic machine, we create a DFA State for every unique set of NFA states the machine could possibly be in.

By tracking sets of states rather than individual states, we eliminate the ambiguity of non-deterministic transitions.

ConceptNFADFA (after conversion)
TransitionsMultiple possible next statesExactly one next state
StatesIndividual NFA statesEach = a set of NFA states
ε-transitionsPresent (free moves)Eliminated (folded into ε-closure)

Stage 1: Establishing Reachability (ε-Closure)

The ε-closure defines all states reachable from a given set using only ε (epsilon) moves. This allows the DFA to account for "hidden" non-deterministic moves before processing any actual input symbol.

ℹ

ε-closure(S) = S ∪ { all states reachable from S via one or more ε-transitions }

We also define move(S, a) as the set of states reachable from any state in S via exactly one transition on input symbol a.

The full transition in the DFA is then: δ_DFA(S, a) = ε-closure( move(S, a) )

Stage 2: Defining the Input Alphabet

The alphabet is all symbols that appear in the NFA's transitions (excluding ε).

Stage 3: Iterative State Discovery

Starting from the initial ε-closure, we build the DFA by finding new state-sets reachable via every alphabet symbol. This process continues until every possible reachable set has been defined as a DFA state.

Stage 4: Completed Deterministic Machine

The final DFA transition table. Every transition is now unique and deterministic. Any DFA state containing an original NFA accept state becomes an accept state in the DFA.

✓

DFA complete. This DFA is equivalent to the original NFA and accepts the same language. Next step: minimize this DFA using Table Filling (Myhill-Nerode).
